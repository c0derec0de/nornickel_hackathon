# SciGraph — Архитектура

Умный помощник для R&D горно-металлургической отрасли: граф знаний + семантический поиск
поверх разнородных документов (статьи, отчёты, протоколы опытов, патенты на RU/EN).

## Ключевая идея

Гибрид **графа знаний (Neo4j)** и **векторного поиска (Chroma)**, оркестрируемый **LLM (Claude)**:
поиск возвращает не только цифры, но и контекст — кто, на каком оборудовании, с каким эффектом.
Две фишки-дифференциатора:

1. **Пробелы в знаниях** — подсветка неизученных комбинаций `материал × режим × условие`.
2. **Противоречия** — конфликтующие числовые/качественные выводы разных источников.

## Онтология (доменная модель)

Типы узлов:
`Material`, `Process`, `Equipment`, `Property`, `Experiment`, `Publication`, `Expert`, `Facility`,
`Condition` (числовое ограничение/диапазон), `Finding` (вывод/эффект), `Geography`.

Ключевые связи:
`USES_MATERIAL`, `OPERATES_AT_CONDITION`, `PRODUCES_OUTPUT`, `DESCRIBED_IN`, `VALIDATED_BY`,
`CONTRADICTS`, `AUTHORED_BY`, `EXPERT_IN`, `PERFORMED_AT`, `HAS_PROPERTY`, `APPLIES_TO`.

Каждый факт несёт метаданные верификации: `source`, `confidence`, `updated_at`, `geo` (RU/world).
Числовые ограничения хранятся структурно: `{quantity, op(<=,>=,range), value, unit}` — чтобы
поддерживать запросы «сульфаты ≤ 300 мг/л», «температура 60–80 °C».

## Компоненты (docker-compose)

```
┌────────────┐   NL-запрос    ┌──────────────────────────────────────┐
│  Frontend  │ ─────────────► │            Backend (FastAPI)          │
│ React+Vite │ ◄───────────── │                                       │
│ chat│graph │  ответ+подграф │  ingest → extract(LLM) → graph+vector │
└────────────┘                │  query  → hybrid retrieve → answer    │
                              │  gaps / contradictions                │
                              └───────┬──────────────────┬────────────┘
                                      │                  │
                               ┌──────▼─────┐     ┌──────▼──────┐
                               │   Neo4j    │     │   Chroma    │
                               │ (граф)     │     │ (вектора)   │
                               └────────────┘     └─────────────┘
```

## Пайплайн импорта

1. **loader** — читает `backend/data/corpus/*` (txt/md/pdf), режет на чанки с метаданными.
2. **extractor (LLM, structured output)** — из чанка достаёт сущности, связи и **числовые
   ограничения** строго в JSON (Claude tool-use, schema-constrained). Нормализация синонимов
   («электроэкстракция»/«electrowinning», «ПВП»/«печь взвешенной плавки»).
3. **graph_builder** — MERGE узлов/связей в Neo4j (идемпотентно), с метаданными верификации.
4. **vector_store** — эмбеддинг чанка (multilingual-e5) → Chroma для семантики.

## Пайплайн запроса (гибрид)

1. **retriever** — семантический поиск релевантных чанков (Chroma) + расширение подграфа
   вокруг найденных сущностей (Neo4j, обход 2–3 уровней) с фильтрами (гео, год, материал,
   числовые диапазоны).
2. **answerer (LLM)** — синтез структурированного ответа с цитатами источников и уровнем
   уверенности; возвращает подграф для визуализации.
3. **gaps / contradictions** — отдельные аналитические запросы к графу.

## Стек

- Backend: Python 3.12, FastAPI, neo4j-driver, chromadb, fastembed (ONNX, без torch)
- LLM: компактная **открытая 7B-модель** (`qwen2.5-7b-instruct`), провайдер настраивается через env:
  **OpenRouter** (OpenAI-совместимый, разрешён в РФ) по умолчанию · **Ollama** локально (приватный) ·
  Anthropic — опционально. Экономия ресурсов: 7B, не 70B, при сохранении точности извлечения.
- Эмбеддинги: `multilingual-e5-large` через fastembed (ONNX, CPU, RU/EN)
- Frontend: React + Vite, react-force-graph-2d, SSE-стриминг ответов
- Инфра: docker-compose (neo4j, chroma, backend, frontend)

## Что заложено, но вне MVP-демо

RBAC/роли, аудит-лог, экспорт PDF/JSON-LD, уведомления, ручная правка графа экспертом,
дашборды руководителя. Архитектура модульная — добавляются как отдельные модули/эндпоинты.
